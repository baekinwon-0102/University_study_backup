파이썬 멀티 스레드와 성능
- 멀티 스레드를 사용해 여러 작업을 동시에 실행하여 처리 속도를 향상시키며,
  작업을 병렬로 처리함으로써 시간을 절약할 수 있음
- 하지만 멀티 스레드를 사용해도 성능 향상이 안됨

파이썬의 GIL
- 파이썬 멀티스레딩은 GIL(Global Interperter Lock)로 인해
  CPU 집중 작업(CPU-bound)에서는 성능 향상이 거의 없거나 오히려 느려질 수 있음
  하지만 네트워크 요청, 파일 I/O 등 대기 시간이 긴 작업(I/O-bound)에서는 스레드 간
  전환을 통해 병렬 처리 효과를 내어 성능이 향상
- 파이썬의 GIL이란, 여러 개의 스레드 중 하나의 스레드만 실행될 수 있도록 하는 
  mutex(상호배제)다
- 파이썬의 garbage collection 메모리 관리 정책을 사용하기 때문
- 단, 스레드가 I/O 작업을 실행하는 동안에는 다른 스레드가 CPU 동작을 동시에 실행할 수
  있음 -> I/O 동작이 많은 프로그램에서는 멀티 스레드의 효과를 얻을 수 있음
