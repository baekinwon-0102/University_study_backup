## 스레드와 소켓 프로그래밍

프로세서와 스레드
- 프로세스: 실행중인 프로그램
- 스레드: 프로세스 안에서 병렬로 실행중인 하위 작업 단위
- 기본적으로 파이썬은 프로세스는 기본적으로 하나의 싱글 스레드에서 동작:
  메인 스레드가 코드를 순차적으로 실행

소켓 blocking I/O
- I/O를 요청한 프로세스는 요청이 완료될 때까지 블록

스레드
- 프로그램은 순차적으로 코드를 실행하기 때문에
- 실행 도중 소켓의 recv(), input()함수와 같은 Blocking function을 만나면
  해당 함수가 실행될 때까지 기다림

멀티 스레드
- 두 개의 함수를 동시에 실행할 수 있게 해주는 것이 멀티 스레드(병렬 실행)
- 둘 이상의 스레드가 동시에 작업 수행

스레드를 이용한 병렬실행
- 별도의 서브 스레드를 생성: 멀티 스레드
- 스레드는 자신이 속한 프로세스의 메모리 공유

스레드 사용법
- threading.Thread() 함수 호출 방식
- threading 모듈의 함수를 이용해 thread 객체 생성 후 start() 호출 방식
- threading.Thread(target = 함수, args = 함수의 매개변수)
  - args는 튜플만 지원하며, 하나의 매개변수를 전달하는 경우, 쉼표를 추가해 튜플로 설정
- 메인 스레드가 종료되어도 서브 스레드는 자신의 작업이 끝날 때까지 계속 실행

메인 스레드가 스레드 종료까지 기다리려면
- 서브 스레드가 종료될 때까지 기다린 후 메인 스레드를 실행하려면 스레드 join() 사용
- join()은 다른 스레드의 종료를 기다린 후 실행

무한루프 스레드의 종료 시점
- 스레드가 한 번 만들어지면 강제 종료시킬 수 있는 방법이 없음
- 메인스레드가 종료될 때 서브 스레드를 종료시키는 방법
  - 스레드를 데몬 스레드로 동작시키는 방법(t.daemon = True)
  - 스레드 이벤트 객체를 사용하는 방법
