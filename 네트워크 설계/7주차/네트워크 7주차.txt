UDP 특징
- 비연결형 서비스, 신뢰성 낮음
- 정보 주고 받을 때 정보 보내거나 받는다는 신호절차 거치지 않음
- 최소한의 오류만 검출
- TCP보다 속도 빠름

UDP 서버 특징
- 연결 자체가 없어서(connect함수 불필요) 서버 소켓과 통신 소켓 구분 없음
- 소켓 대신 IP 기반으로 데이터 전송
- 서버와 클라이언트 1대1, 1대N, N대M 등으로 연결 가능
- 신뢰성 필요한 서비스보다 성능 중요시 되는 경우 사용

TCP와 UDP 비교
- 사진

UDP 소켓 서버 클라이언트 특징
- UDP에서 서버와 클라이언트는 연결되어있지 않음
  - TCP와 달리 연결 설정 과정 불필요 따라서 listen함수와 accept함수 호출 없음
    다만 UDP 소켓의 생성과 데이터 송수신 과정만 존재
- UDP에선 서버와 클라이언트 모두 하나의 소켓만 있으면 됨
  - 예를 들어 10개 클라이언트와 통신 위해선 TCP는 통신 소켓 10개 필요하지만
    UDP는 서버건 클라이언트건 하나의 소켓만 있으면 됨
- UDP 소켓에서 데이터 전송 시 상대방 주소정보 별도 추가
  - TCP 소켓 생성 후 데이터 전송 시, 상대방 주소 정보 추가하는 과정 불필요
  - 그러나 UDP 소켓은 연결상태 유지하지 않으므로 데이터 전송할 때마다 반드시
    목적지 주소 정보를 별도로 추가해야 함 -> 수신 시 return 값 두 개(메시지, 주소정보)

UDP 소켓통신 절차
- 연결설정과정 없음
- 서버는 UDP 소켓 1개만 사용
- 데이터 전송 시 상대방 주소정보 추가
- 데이터 수신 시 상대방 주소정보 반환

UDP 파일 전송
- 안정성이 떨어지나 데이터 더 빠른 속도로 전송하는데 사용하므로 오디오, 비디오 파일과
  같은 데이터 전송 시 사용

UDP 소켓 이용한 에코서버와 클라이언트
- 에코서버: 클라이언트가 보낸 데이터 서버에서 받아 그대로 클라이언트에게 돌려보내는
  네트워크 서버
- 스레드 사용 안하고 한 소켓으로 여러 클라이언트 처리 가능
- 클라이언트는 별도 연결설정 과정없이 서버와 통신
- UDP 서버는 하나의 소켓 이용해 다수 클라이언트와 통신 가능
