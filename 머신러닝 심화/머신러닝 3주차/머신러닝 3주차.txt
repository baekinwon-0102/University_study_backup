클러스터와 클러스터링
- 클러스터 (Cluster): 유사한 여러 개의 클래스로 나누어진 데이터
- 클러스터링: 유사한 특성을 가진 그룹들로 묶는 작업
- 같은 클러스터의 것은 다른 클러스터의 것보다 더 유사
- 이와 같은 유사한 것들끼리의 집합이 바로 클러스터

분류와 클러스터링의 차이점
- 분류는 지도 학습 영역, 클러스터링은 비지도 학습 영역  ★ 문제 나올 확률 높음 ★
- 분류는 라벨이 정해져 있는 데이터를 기준에 따라 구분하는 것
- 클러스터링은 유사성에 따라 몇 개의 클러스터들로 묶는 것
  - 보다 복잡한 처리작업 할 수 있음
  - 데이터의 근본 구조를 확인할 수 있음

K-means 클러스터링
- 'K-평균 군집화' 라고도 함
- 비지도 학습 알고리즘 중 대표적인 클러스터링 방법
- 간단하면서도 많이 쓰이는 클러스터링 방법 중 하나
- 유사한 특성을 가진 k개의 데이터 그룹으로 묶는 방법
- 라벨이 없는 원데이터 -> 각기 다른 특성을 지닌 2개의 대표 데이터 선정 -> 
  대표 데이터와 유사하게 보이는 데이터를 묶음
- 주어진 데이터 집합에 대해 k개의 클러스터 중심점 찾기
- 각 클러스터에는 클러스터 중심이 있음
- 각 점은 다른 중심점보다 지정된 클러스터 중심점에 더 가까움

분석 진행 절차  ★ 시험 문제 출제 가능성 높음 ★
- 1) 몇개의 클래스로 분류할 것인지 설정
  - K-평균 알고리즘의 K는 몇 개의 클래스로 분류할 것인지를 나타내는 변수
- 2) 클러스터의 최초 중심 설정
  - 클러스터의 최초 중심을 무작위로 설정
  - 클러스터의 최초 중심점을 지정
  - 최초 데이터 포인트, 해당 포인트로부터 가장 먼 포인트 순으로 지정
- 3) 데이터를 가장 가까운 클러스터로 지정
- 4) 클러스터 중심을 클러스터에 속한 데이터들의 가운데 위치로 변경
- 5) 클러스터 중심이 바뀌지 않을 때까지 3,4번 과정 반복

장단점
- 장점
  - 알고리즘이 비교적 간단하고, 수행 속도가 빠름
  - 주어진 데이터에 대한 사전 정보 없이 클러스터링을 함
  - 데이터를 분류하는 머신러닝과 데이터 마이닝의 도구

- 단점
  - 클러스터링의 개수 k와 최초로 지정하는 중심점들에 따라 결과가 다소 달라질 수 있음

활용 분야
- 통계: 주어진 데이터의 분류나 성향 분석
- 전자상거래: 고객의 구매 이력으로 고객 분류
- 건강 관리: 질병과 치료를 위한 패턴 탐지
- 패턴: 유사한 이미지 그룹화
- 재무: 신용카드 사기 탐지
- 회사: 매출 등을 토대로 회사의 등급 분류
- 기술: 네트워크 침입과 악의적 활동 탐지
- 기상 예보: 폭풍 예측

계층적 클러스터링 ★ 시험 문제 출제 가능성 높음 ★
- 병합형/응집형/상향식 계층 분류법(agglomerative method)
  - 개별 데이터 중 가장 가까운 두 데이터를 하나의 그룹으로 결합
  - 처음 N개의 데이터에서 N-1개가 된 데이터 중 다시 가장 가까운 데이터 결합
  - 반복시행함

- 분리형/분할형/하향식 계층분류법(divisive)
  - 전체 데이터를 우선 두 개로 분류
  - 분류된 두 집단 중 하나를 선택해 다시 두 개로 분류
  - 반복 시행

유사도 계산법 ★ 시험 문제 출제 가능성 높음 ★
- 단순 링크법 (single link): 두 클러스터 중 가장 가까운 두 데이터 사이 유사도로 측정
- 완전 링크법 (complete link): 두 클러스터 중 가장 먼 두 데이터 사이 유사도로 측정
- 집합 평균법 (average link): 두 클러스터 평균값으로 측정
