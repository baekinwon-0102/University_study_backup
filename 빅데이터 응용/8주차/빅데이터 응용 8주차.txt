## 분석 프로젝트 관리 방안

분석 과제 5가지 주요 특성 및 관리 영역
- 데이터 크기: 분석하고자 하는 데이터 양 고려한 관리 방안 수립 필요
- 데이터 복잡성: 텍스트, 오디오, 비디오 등 비정형 데이터 및 다양한 시스템에 산재된
  원천 데이터 통합해 분석 프로젝트 진행 시 초기 데이터 확보, 통합 및 분석 모델 선정
  등 사전 고려 필요
- 속도: 분석 결과 도출 시 이를 활용하는 시나리오 측면에서 속도 고려
  일단위, 주단위 실적 경우 배치 형태로 작업 가능하지만 고객에게 개인화된 상품/서비스
  추천 시 분석 모델 적용 및 계산이 실시간으로 수행 필요해 분석모델 속도 고려해야 함
- 분석 복잡성: 분석 모델 정확도와 복잡도는 트레이드 오프 관계 존재, 모델 복잡할 수록
  정확도 올라가지만 해석 어려워지는 단점 존재 -> 이에 대한 기준점 사전에 정의 필요
  따라서 해석 가능하면서도 정확도 올릴 수 있는 최적모델 찾는 방안 사전 모색 필요
- 정확도와 정밀도: 정확도는 모델과 실제 값 사이 차이 적은 정도 의미 정밀도는 모델
  지속적으로 반복 시 편차 수준으로 일관적으로 동일한 결과 제시하는 것 의미
  분석 활용 측면에선 정확도가 중요, 안정성 측면에선 정밀도 중요

분석 프로젝트 특성
- 분석가 입장에서 데이터 원처 다루는 데이터 영역과 결과 활용할 비즈니스 영역의 중간에서
  분석 모델 통한 조율 수행하는 조정자 역할이 핵심
- 특히 분석 프로젝트에서 데이터 영역과 비즈니스 영역의 현황 이해, 프로젝트 목표인 분석
  정확 도달성과 결과에 대한 가치 이해 전달하는 조정자로써 분석가 역할 중요
- 조정자로써 분석가가 해당 프로젝트 관리자까지 겸임하게 되는 경우 대부분 -> 
  프로젝트 관리 방안에 대한 이해와 주요 관리 포인트 사전 숙지 필수

분석 프로젝트 관리
- 데이터 분석 특성 살려 프로젝트관리 지침인 KS AISO 21500:2013을 기본 가이드로 활용
- 프로젝트관리 지침의 프로젝트 관리 체계는 통합, 이해관계자, 범위, 자원, 시간, 원가, 
  리스크, 품질, 조달, 의사소통의 10개 주체 그룹 구성

KS AISO 21500 프로젝트 관리 주체 그룹
- 통합: 프로젝트 관련된 다양한 활동과 프로세스 도출, 정의, 결합, 단일화, 조정, 통제,
   종료에 필요한 프로세스
- 이해관계자: 프로젝트 관계자, 고객사, 기타 이해관계자 식별하고 관리
- 범위: 작업과 결과물 식별하고 정의
- 자원: 인력, 시설, 장비, 자재, 기반시설, 도구와 같은 적절한 프로젝트 자원 식별하고 확보
- 시간: 프로젝트 활동 일정 수립하고 일정 통제 진척상황 관찰하는데 요구
- 원가: 예산 개발과 원가통제의 진척상황 관찰하는데 요구
- 리스크: 위험과 기회 식별하고 관리
- 품질: 품질보증과 품질통제 계획하고 확립
- 조달: 계획에 요구된 프로세스 포함해 제품 및 서비스 또는 인도물 인수하고 공급자와의
  관계 관리
- 의사소통: 프로젝트와 관련된 정보 계획, 관리, 배포
