빅데이터의 유형
- 데이터의 구조적 관점에서 정형, 반정형, 비정형 데이터로 구분
- 정형 데이터
  - 정형화된 구조, DBMS에 내용이 저장될 수 있는 구조
  - 고정된 필드에 저장된 데이터
  - 예시) 엑셀, 관계형 DB
- 반정형 데이터
  - 데이터 내부에 데이터 구조에 대한 메타 정보가 포함된 구조
  - 고정된 필드에 저장되어 있지만 메타데이터나 데이터 스키마 정보를 포함하는 데이터
  - 예시) 날씨 데이터, 모바일 데이터, HTML, JSON 등
- 비정형 데이터
  - 수집 데이터 각각이 데이터 객체로 구분
  - 고정 필드 및 메타데이터가 정의되지 않음
  - 예시) 이미지, 음성, 동영상, QR코드 등

빅데이터로 인한 본질적 변화
- 사전처리 -> 사후처리
  - 사전처리는 필요한 정보만 수집하고 필요하지 않은 정보 버림
  - 이미 가치가 있을 것이라고 정해진 특정한 정보만 모아서 처리하는 것이 아니라
    가능한 많은 데이터를 모으고 그 데이터를 다양한 방식으로 조합
- 표본조사 -> 전수조사
  - 표본조사는 기본적으로 분석하고자 하는 바를 정하고 그에 필요한 정보 수집
    때문에 질문 바뀌면 다시 수집해야함
  - 전수조사의 경우 모든 데이터를 모아 놓기때문에 바뀐 질문에 따라 다양한 방식으로
    데이터를 재가공할 수 있음
- 질 -> 양
  - 번역 프로그램 개발 중 IBM은 질을 중시했고 구글은 양을 중시했는데 구글이 살아남음
- 인과관계 -> 상관관계

빅데이터의 가치
- 빅데이터의 가치 선정이 어려운 이유
  - 데이터의 재사용이나 재조합, 다목적용 데이터 개발 등이 일반화되면서 특정 데이터를
    언제 어디서 누가 활용할지 알 수 없음
  - 다목적용 데이터 개발의 사례(CCTV로 수집한 데이터는 절도범 뿐 아니라 고객의 구매
    정보도 동시에 얻을 수 있음)
  - 기존에 없던 가치를 창출함에 따라 그 가치 측정하기 어려움
  - 이러한 정보를 저자에게 주는 가치는 이전에 없던 것으로 어느 정도의 가치가 있는지
     측정하기 어려움
  - 분석기술이 발달함에 따라 데이터의 가치에 영향 줌

빅데이터의 영향
- 빅데이터가 가치를 만들어 내는 방식
  - 투명성 제고로 연구개발 및 관리 효율성 제고
  - 시뮬레이션을 통한 수요 포착 및 주요 변수 탐색으로 경쟁력 강화
  - 고객 세분화 및 맞춤 서비스 제공
  - 알고리즘을 활용한 의사결정 보조 혹은 대체
  - 비즈니스 모델과 제품, 서비스의 혁신
- 기업
  - 소비자 행동 분석, 시장 변동 예측해 비즈니스 모델을 혁신하거나 신사업 발굴 가능
  - 원가절감, 제품 차별화, 기업활동의 투명성 제고 등에 활용하면 경쟁사 대비 강한
    경쟁력 확보 가능
  - 빅데이터의 활용으로 기업들의 운용 효율성이 증가하면 산업 전체의 생산성 향상되고
   국가 전체로서는 GDP가 올라감
- 정부
  - 각종 데이터 수집해 사회 변화를 추정하고 각종 재해 관련 정보를 추출할 수 있음
  - 수집된 데이터 바탕으로 사회관계망 분석이나 시스템 다이내믹스, 복잡계 이론과 같은
   분석방식을 적용해 미래 의제를 도출할 수 있음
   ※시스템 다이내믹스: 사업이나 사회 시스템 등과 같은 복잡한 시스템을 연구하고 관리
                              하는 방법
  - 도출한 미래 의제에 대한 대응 방안 역시 빅데이터를 통해 얻을 수 있음
- 개인
  - 빅데이터 활용대상의 위치에 머물러 있지만 빅데이터를 서비스로 제공하는 기업들이
    출현해 적시에 필요한 정보를 획득할 수 있으며 저렴한 비용으로 활용 가능

소셜네트워크 분석의 측정지표
- 연결정도
  - 노드 간의 총 연결 관계 개수
  - 한 노드가 몇 개의 노드와 연결되어 있는지의 정도
- 포괄성
  - 네트워크 내에서 서로 연결된 노드의 개수
- 밀도
  - 네트워크 내에서 노드 간의 전반적인 연결 정도 수준을 나타내는 지표
  - 연결망 내 전체 구성원이 서로 간에 얼마나 많은 관계를 맺고 있는지 표현
- 중심성
  - 노드의 상대적 중요도를 나타내는 척도
  - 연결정도 중심성: 특정 노드가 연결망내에서 직접 연결된 노드 또는 링크의 수
    - 해당 노드와 직접 연결된 노드의 개수/ 해당 노드와 직간접적으로 연결된 노드의 수
  - 근접 중심성: 각 노드 간의 거리를 바탕으로 중심성을 측정하는 방식
    - 해당 노드와 직간접적으로 연결된 노드의 수-1/해당노드와 다른노드들 간 최단거리 합
  - 매개 중심성: 특정 노드가 다른 노드들 사이에 위치하는 정도 나타냄
    - 해당 노드를 통과하는 최단 경로들의 개수
  - 위세 중심성: 자신의 연결정도 중심성으로부터 발생하는 영향력과 자신과
    연결된 타인의 영향력을 합해 결정하는 방법

  - 빅데이터 기반 분석, 예측 기술 발전으로 인한 정확도 증가로 분석 대상이 되는 사람들은
    예측 알고리즘의 희생양 될 가능성 올라감
  - 미국 경잘관들의 경우 컴퓨터 알고리즘 분석에 따라 특정지역을 순찰해 강력범죄 발생률
   을 상당수 감소 시켰으나 이를 조금 더 진전시키면 범죄 예측 프로그램에 의해 범행을
   저지르기 전 체포될 수도 있음
  - 특정인이 빅데이터 분석 결과에 따라 특정한 행위를 할 가능성이 높다는 이유만으로
    처벌 받는 것으로 행위 결과에 대해서만 책임을 묻는다는 민주주의 사회 원칙을 훼손할 
    수 있음
  - 어떤 사람이 특정한 사회, 경제적 특성을 가진 집단에 속한다는 이유로 자신의 신용도와
   무관하게 부당하게 대출이 거절되는 상황도 알고리즘 예측의 피해 사례가 될 수 있음
- 데이터의 오용
  - 베트남 전쟁 때 맥나마라는 적군 사망자 수를 전쟁의 진척 상황을 나타내는 지표로 활용
  - 그 결과 적군 사망자 수는 과장되어 보고되는 경향이 강화되었으며 언론에 공개된 적군
   사망자 수는 결과적으로 전쟁상황을 오도하는 결과 가져옴
  - 주어진 데이터에서 잘못된 인사이트를 얻어 비즈니스에 적용할 경우 손실을 불러올 수 
    있는데 예를 들어 구글이 검색 알고리즘에 잘못된 시그널을 사용할 경우 특정 거래
   사이트가 검색 상단에서 밀려나 매출 급감해 시장에서 퇴출 가능성 있음

- 사생활침해 -> 책임제의 시행
  - 사용자 정보는 수집된 후 한 번 사용하고 폐기되는 것이 아니라 1차적 목적 외에
    2차, 3차적 목적으로 가공, 유통, 활용될 수 있음
  - 개인정보의 활용에 대해 개인이 매번 동의하는 것은 경제적으로도 매우 비효율적이며
    가능하지도 않은 상황이 발생할 수 밖에 없음
  - 따라서 사생활침해 문제를 '개인정보 제공자의 동의'를 통해 해결하기 보다 '개인정보
    사용자의 책임'으로 해결하는 것이 필요
- 책임원칙 훼손 -> 결과 기반 책임 원칙
  - 특정 기업이 담합할 가능성이 높다고 판단한 예측 알고리즘의 판단을 근거로 해당
    기업을 처벌하면 안되며 실제 담합한 결과에 대해 처벌해야 함
  - 특정인이 채용이나 담보대출, 신용카드 발급 여부 결정 등에서 예측 자료에 의해 불이익
    을 당할 가능성을 최소화하는 장치 마련해야 함
- 데이터 오용 -> 알고리즘에 대한 접근권 제공
  - 알고리즘의 객관적 인증방안을 도입하자는 의견도 제시되고 있으며 알고리즘의 부당함
   을 반증할 수 있는 방법을 명시해 공개할 것을 주문하기도 함
  - 불이익을 받는 사람들을 대변해 피해자를 구제할 수 있는 능력을 가진 전문가가 필요

미래의 빅데이터
- 빅데이터 활용에 따라 필요한 기본적 3요소는 데이터, 기술, 인력임
- 데이터 분야
  - 데이터 수집을 위한 다양한 기기 관련 특허를 출원 중이며, 자동차는 센서의 집합체로
    변신중이며 각 가정은 각종 센서들이 온도와 전기를 제어하는 시대로 접어들고 있음
  - 모든 것들이 데이터화하는 시대가 피할 수 없는 물결로 밀려오고 있으며 센서로부터
    수집되는 데이터는 기존 기업의 비즈니스 모델을 바꾸기도 함
  - 핵심 비즈니스 모델이 제조업에서 서비스업으로 변화하고 있음
- 기술 분야
  - 빅데이터 분석 알고리즘의 진화가 빠르게 진행
  - 알고리즘은 데이터 양 증가에 따라 정확도가 증가하는 일반적인 경향이 있는데
   M2M, IoT의 확산으로 데이터 생산량이 기하급수적으로 증가함에 따라 빅데이터를
   다루는 알고리즘의 효율성이 기하급수적으로 증가할 것
  - 구글은 자기학습이 가능한 인공신경망 개발에 성공
  - 스스로 학습이 가능한 인공지능 기술은 기존의 데이터 분석기법으로 처리가 불가능할
   만큼 거대한 규모의 빅데이터 처리에 필수 불가결한 기술
- 인력 분야
  - 데이터 사이언티스트는 빅데이터에 대한 이론적 지식과 숙련된 분석 기술을 바탕으로 
    통찰력, 전달력, 협업능력을 두루 갖춘 전문인력을 의미
  - 데이터 사이언티스트는 빅데이터의 다각적 분석을 통해 인사이트 도출하고 이를
   조직의 전략 방향 제시에 활용할 줄 아는 기획자로서 전문가 역할을 할 것으로 기대됨
  - 알고리즈미스트는 데이터 사이언티스트가 한 일로 인해 부당하게 피해 발생하는 것을
    막기 위해 필요
  -알고리즈미스트 역시 컴퓨터, 수학, 통계뿐만 아니라 비즈니스 전반에 대한 이해가 필요
   하고 알고리즘 코딩 해석을 통해 빅데이터 알고리즘에 의해 부당하게 피해를 입은 사람들
  을 구제하는 전문가
