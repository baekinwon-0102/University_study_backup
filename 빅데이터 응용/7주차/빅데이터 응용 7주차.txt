## 분석마스터 플랜 수립

분석 마스터 플랜 수립 프레임워크
- 기업 및 공공기관에서 시스템의 중장기 로드맵을 정의하기 위해
  정보전략계획인 ISP 수행
- ISP는 정보기술 또는 정보시스템을 전략적으로 활용하기 위해 조직 내-외부 환경을
  분석해 기회나 문제점을 도출하고 사용자의 요구사항을 분석하여 시스템 구축 우선순위를
  결정하는 등 중장기 마스터플랜을 수립하는 절차
- 중-장기적 관점의 마스터플랜 수립 위해서는 분석과제를 대상으로 전략적 중요도, 
  비즈니스 성과 및 ROI, 분석과제의 실행 용이성 등 다양한 기준을 고려해 적용할
  우선순위 설정

분석 마스터플랜 수립 기준
- 일반적인 IT 프로젝트의 우선순위 평가는 전략적 중요도와 실행 용이성으로 나눌 수 있음

전략적 중요도
- 전략적 필요성
  - 전략적 목표 및 본원적 업무에 직접적인 연관관계가 일정한 정도
  - 이슈 미해결 시 발생하게 될 위험 및 손실에 대한 정도
- 시급성
  - 사용자 요구사항, 업무 능률 향상을 위해 시급히 수행되어야 하는지에 대한 정도
  - 향후 경쟁우위 확보를 위한 중요성 정도

실행 용이성
- 투자 용이성
  - 기간 및 인력 투입 용이성 정도
  - 비용 및 투자예산 확보 가능성 정도
- 기술 용이성
  - 적용 기술의 안정성 검증 정도
  - 응용 시스템, H/W 유지보수 용이성 정도
  - 개발 스킬 성숙도 및 신기술 적용성 정도

분석 마스터 플랜 수립 프레임워크2
- 데이터 분석 과제의 우선순위 평가기준은 기업이 당면한 상황에 따라 다르므로 기존의
  IT 프로젝트의 우선 순위 평가 기준과 다른 관점에서 살펴볼 필요가 있음
- 빅데이터의 특징(4V, Volume, Variety, Velocity, Value)을 고려한 분석 ROI요소는 다음과
  같음
- 3V(Volume, Variety, Velocity)는 투자비용, Value는 비즈니스 효과 측면에서 
  검토할 수 있음
  - 데이터 크기: 데이터의 규모 및 양을 의미, 대용량 데이터를 저장-처리하고 관리하기
    위해 새로운 투자 필요
  - 데이터 다양성: 데이터의 유형, 데이터 소스(내외부 등)를 입수하는데 있어 투자가 필요
  - 데이터 속도: 데이터 처리속도로 데이터를 빠르게 가공 분석하는 기술에 투자 필요
  - 데이터 가치: 분석결과를 활용하거나 실질적인 실행을 통해 얻게 되는 비즈니스 효과

수행 과제 도출 및 우선순위 평가
- 정의된 데이터 과제에 대한 실행순서 정하는 방법
- 기준에 따라 평가 후 과제의 선후 관계를 고려해 적용 순위 확정
- 우선순위 평가 기준에 시급성과 난이도가 있음

시급성
- 전략적 중요도와 목표 가치에 부합하는지에 따른 시급성이 가장 중요한 기준
- 시급성의 판단 기준은 전략적 중요도가 핵심사항
- 분석과제의 목표 가치와 전략적 중요도를 현재의 관점에 둘 것인지, 미래의 관점에
  둘 것인지를 함께 고려해 시급성 여부를 판단해야 함

난이도
- 현재 기업의 분석 수준과 데이터를 생성, 저장, 가공, 분석하는 비용을 고려한 난이도는
  중요한 기준
- 난이도는 현재 시점에서 과제를 추진하는 것이 범위 측면과 비용 측면에서 바로 적용하기
  쉬운 것인지 어려운 것인지에 대한 판단 기준으로 데이터 분석의 적합성 여부의 기준이 됨

포트폴리오 사분면 분석을 통한 과제 우선순위 선정 기법
- 1사분면: 전략적 중요도가 높아 시급하게 추진이 필요하나, 난도가 높아 바로 적용 어려움
- 2사분면: 전략적 중요도가 높지 않지만 중장기적 관점에서 반드시 추진 필요, 난도가 높음
- 3사분면: 전략적 중요도가 높고, 난도가 어렵지 않아 우선적으로 바로 적용
- 4사분면: 전략적 중요도가 높지 않고, 과제 적용이 어렵지 않음

수행 과제 도출 및 우선순위 평가
- 사분면 영역에서 가장 우선적인 분석 과제 적용이 필요한 영역은 3사분면 영역
- 전략적 중요도가 낮은 편이지만 중장기적으로 경영에 미치는 영향도가 높고
  분석 과제를 바로 적용하기 어려워 우선순위가 낮은 영역은 2사분면 영역
- 분석 과제의 적용 우선순위 기준을 '시급성'에 둔다면 3->4->2 영역 순,
  '난이도'에 둔다면 3->1->2 영역 순

이행계획 수립 -> 로드맵 수립
- 과제의 우선순위를 토대로 분석과제별 적용범위 및 방식을 고려하여 최종적인 실행 
  우선순위를 결정한 후 단계적 구현 로드맵 수립
- 단계적 구현 로드맵은 추친단계, 단계별 추진목표, 추진과제로 나열 가능

세부 이행계획 수립
- 데이터 분석 체계는 일반적으로 데이터 수집 및 확보와 분석 데이터를 준비하는 단계를
  순차적으로 수행하고 모델링 및 평가 단계는 반복적으로 수행하는 혼합형 주로 적용
- 데이터 분석 과제 정의
  - 프로젝트 전체 일정에 맞게 사전 준비를 하는 단계
  - 단계별 필요 산출물, 주요 보고 시기 등으로 구분해 세부 단위별 일정과 전체 일정이
    예측될 수 있도록 일정 수립
- 데이터 준비 및 탐색
  - 데이터 처리 엔지니어와 데이터 분석가의 역할을 구분해 세부 일정이 만들어 지는 단계
  - 데이터 처리 엔지니어가 필요 데이터를 수집하고 정리하는 일정 수립
  - 데이터 분석가가 분석에 필요한 데이터들로부터 변수 후보를 탐색하고 최종적으로 도출
   하는 일정 수립
- 데이터 분석 모델링 및 검증
  - 데이터 준비 및 탐색이 완료된 이후 데이터 분석 가설이 증명된 내용을 중심으로 데이터
    분석 모델링을 진행하는 단계
  - 데이터 분석 모델링 과정에 대해서 실험방법 및 절차 구분
  - 기획하고 검증하는 내용에 대해 자세한 일정을 수립
- 산출물 정리
  - 데이터 분석단계별 산출물을 정리하고, 분석 모델링 과정에서 개발된 분석 스크립트
    등을 정리하여 최종 산출물로 정리하는 단계

## 분석 거버넌스 체계 수립

거버넌스 체계 개요
- 데이터 분석을 기업의 문화로 정착하고 데이터 분석 업무를 지속적으로 고도화하기 위해
  조직 내 분석 거버넌스(공동의 관심사에 대한 네트워크를 구축해 문제를 해결하는 방식)
  체계를 수립해야 함
- 데이터 분석의 지속적인 적용과 확산을 위한 분석 거버넌스 체계 구성요소는 다음과 같음
  - 시스템: 분석관련 IT 시스템 및 프로그램
  - 인적 자원: 분석관련 교육 및 마인드 육성 체계
  - 조직: 분석기획 및 관리 수행 조직
  - 데이터: 데이터 거버넌스
  - 프로세스: 과제 기획 및 운영 프로세스

데이터 분석 성숙도 모델 및 수준진단
- 6개 영역에서의 분석 준비도와 3개 영역에서 분석 성숙도를 평가해 수행
- 분석을 위한 준비도 및 성숙도를 진단하는 궁극적 목표는 각 기업이 수행하는 현재의
  분석 수준을 명확히 이해하고 수준 진단 결과를 토대로 미래의 목표수준을 정의하는데
  있음

분석 준비도
- 분석 준비도는 기업의 데이터 분석 도입의 수준을 파악하기 위한 진단방법으로 6가지
  영역을 대상으로 현 수준 파악
  - 분석업무 파악
    - 발생한 사실, 예측, 시뮬레이션, 최적화 분석 업무
    - 분석업무 정기적 개선
  - 인력 및 조직
    - 분석 전문가 교육 훈련 프로그램, 관리자들의 기본적 분석 능력
    - 전사 분석업무 총괄 조직 존재, 경영진 분석업무 이해 능력
  - 분석기법
    - 업무별 적합한 분석기법 사용, 분석업무 도입 방법론
    - 분석기법 라이브러리, 분석기법 효과성 평가 및 정기적 개선
  - 분석 데이터
    - 분석업무를 위한 데이터 충분성/신뢰성/적시성
    - 비구조적 데이터 관리, 외부 데이터 활용 체계, 기준 데이터 관리
  - 분석 문화
    - 사실에 근거한 의사결정, 관리자의 데이터 중시, 회의 등에서 데이터 활용
    - 경영진의 직관보다 데이터 활용, 데이터 공유 및 협업 문화
  - IT 인프라
    - 운영시스템 데이터 통합, EAI(기업어플리케이션통합), ETL(데이터 추출/변환/적재)
      등 데이터 유통체계
    - 분석 전용 서버 및 스토리지
    - 빅데이터 분석환경/통계 분석환경/비주얼 분석환경
- 진단결과 전체 요건 중 일정 수준 이상 충족하면 분석업무를 도입하고 충족 못하면
  먼저 분석환경 조성

데이터 분석 성숙도 모델 및 수준진단(분석 성숙도)
- CMMI, 능력성숙도 통합 모델을 기반으로 분석 능력 및 분석 결과 활용에 대한 조직의
  성숙도 수준을 평가해서 현재 상태를 점검함
- 조직평가를 위한 성숙도 단계는 4단계로 도입, 활용, 확산, 최적화 단계로 되어있음
- 분석 성숙도 진단은 비즈니스, 조직-역량, IT 부문 등 3개 부문을 대상으로 성숙도 수준
  평가

데이터 분석 성숙도 모델 및 수준진단(분석 수준 진단 결과)
- 기업의 현재 분석 수준을 객관적으로 파악하고 경쟁사의 분석 수준과 비교해
  분석 경쟁력 확보 및 강화를 위한 목표 수준을 설정할 수 있음
- 성숙도와 준비도에 따라 4가지 유형으로 구분해 분석 수준에 대한 목표방향을 정의하고
  유형별 특성에 따라 개선방안 수립
- 준비형: 해당 위치의 기업들은 분석을 위한 데이터, 조직 및 인력, 분석 업무, 분석 기법
  등이 적용되지 않음으로 인해 사전 준비가 필요한 유형
- 정착형: 준비도는 낮은 편이지만, 조직, 인력 분석업무, 분석 기법 등을 기업 내부에서 
  제한적으로 사용하고 있어 분석의 정착이 필요한 기업 유형
- 도입형: 기업에서 활용하는 분석업무 및 기법 등은 부족한 상태지만 조직 및 인력 등
  준비도가 높은 유형으로 데이터 분석을 바로 도입할 수 있는 기업
- 확산형: 데이터 분석을 위해 기업에 필요한 6가지 분석 구성요소를 모두 갖추고 있으며
  현재 부분적으로 도입해 지속적인 확산이 가능한 기업

분석 지원 인프라 방안 수립
- 분석과제 단위별로 별도의 분석 시스템 구축하는 경우 관리의 복잡도 및 비용 증대라는
  부작용이 나타남
- 분석 마스터플랜을 기획하는 단계부터 장기적이고 안정적으로 활용할 수 있는 확장성
  고려한 플랫폼 구조를 도입하는 것이 필요

인프라 구축 유형
- 개별 시스템
  - 구조: 네트워크 형
  - 시스템 간 자체적인 데이터 교환
  - 시스템별 독립적인 데이터 관리
  - 확장 시 시스템 간 인터페이스 폭증

## 데이터 거버넌스 체계 수립

데이터 거버넌스 체계 수립
- 데이터 거버넌스는 전사 차원의 모든 데이터에 대해 정책 및 지침, 표준화, 운영조직 및
  책임 등의 표준화된 관리 체계를 수립하고 운영을 위한 프레임워크 및 저장소를 구축하는
  활동을 의미
- 마스터 데이터, 메타 데이터, 데이터 사전은 데이터 거버넌스의 중요한 관리 대상
  - 메타 데이터: 데이터에 관한 구조화된 데이터로 다른 데이터를 설명해주는 데이터
  - 데이터 사전: 데이터에 관한 정보를 수집, 보관, 제공하기 위한 장치
- 기업은 데이터 거버넌스 체계를 구축함으로써 데이터 가용성, 유용성, 통합성, 보안성,
  안정성을 확보할 수 있음
- 데이터 거버넌스는 독자적으로 수행될 수도 있지만 전사 차원의 IT 거버넌스와 EA(종합
  설계도)의 구성요소로 구축되는 경우도 있음
- 빅데이터 거버넌스는 데이터 거버넌스의 체계에 더해 빅데이터의 효율적인 관리, 다양한
  데이터의 관리체계, 데이터 최적화, 정보보호, 데이터 생명주기 관리, 데이터 카테고리별
  관리 책임자 지정등을 포함
- 데이터 거버넌스의 구성요소인 원칙, 조직, 프로세스는 유기적인 조합 통해 데이터를
  비즈니스 목적에 부합하고 최적의 정보 서비스를 제공할 수 있도록 효과적으로 관리
  - 원칙
    - 데이터를 유지-관리하기 위한 지침과 가이드
    - 보안, 품질기준, 변경관리
  - 조직
    - 데이터를 관리할 조직의 역할과 책임
    - 데이터 관리자, 데이터베이스 관리자, 데이터 아키텍트 등
  - 프로세스
    - 데이터 관리를 위한 활동과 체계
    - 작업 절차, 모니터링 활동, 측정활동 등

데이터 거버넌스 체계
- 데이터 거버넌스 체계는 데이터 표준화, 데이터 관리 체계, 데이터 저장소 관리, 
  표준화 활동으로 나누어 볼 수 있음
- 데이터 표준화
  - 데이터 표준용어 설정, 명명규칙, 메타데이터 구축, 데이터 사전 구축 등의 업무로 구성
  - 데이터 표준 용어는 표준 단어 사전, 표준 도메인 사전, 표준 코등 등으로 구성되며 사전
    간 상호 검증이 가능하도록 점검 프로세스를 포함해야 함
- 데이터 관리 체계
  - 데이터 정합성 및 활용의 효율성을 위해 표준 데이터를 포함한 메타 데이터와
    데이터 사전의 관리 원칙을 수립
  - 수립된 원칙에 근거해 항목별 상세한 프로세스를 만들고 관리와 운영을 위한 담당자 및
    조직별 역할과 책임 준비
  - 빅데이터의 경우 데이터 양의 급증으로 데이터의 생명주기 관리방안을 수립
- 데이터 저장소 관리
  - 메타 데이터 및 표준 데이터를 관리하기 위한 전사 차원의 저장소 구성
  - 데이터 저장소는 데이터 관리체계 지원을 위한 워크플로우 및 관리용 응용소프트웨어를
    지원하고 관리 대상 시스템과의 인터페이스를 통한 통제가 이루어져야 함
- 표준화 활동
  - 데이터 거버넌스 체계 구축 이후 표준 준수 여부를 주기적으로 점검 및 모니터링 시행

데이터 분석 조직 개요
- 데이터 분석 조직은 기업의 경쟁력을 확보하기 위해 데이터 분석의 가치를 발견하고 이를
  활용해 비즈니스를 최적화하는 목표를 갖고 구성되어야 함
- 이를 위해 기업의 업무 전반에 걸쳐 다양한 분석과제를 발굴해 정의하고 데이터 분석을
  통해 의미있는 인사이트를 찾아 실행하는 역할을 수행할 수 있어야 함

데이터 조직 및 인력방안 수립
- 데이터 분석 전문조직은 조직구조 및 인력구성을 고려해 기업에 최적화 된 형태로 구성
- 조직 구조
  - 비즈니스 질문을 선제적으로 찾아 낼 수 있는 구조인지
  - 분석 전담조직과 타 부서간 유기적인 협조와 지원이 원활한 구조인지
  - 효율적인 분석 업무를 수행하기 위한 분석 조직 구조인지
  - 전사 및 단위부서가 필요 시 접촉하며 지원할 수 있는 구조인지
- 인력 구성
  - 비즈니스 및 IT 전문가의 조합으로 구성되어야 하는지
  - 어떤 경험과 어떤 스킬을 갖춘 사람으로 구성해야 하는지
  - 통계적 기법 및 분석 모델링 전문인력을 별도로 구성해야 하는지
  - 전사 분석업무에 대한 적합한 인력 규모는 어느 정도인지
- 데이터 분석을 위한 조직구조는 분석업무 수행 주체에 따라 3가지 유형의 조직구조로
  살펴볼 수 있음
  - DsCoE: 데이터 사이언스 전문가들로 구성된 조직
- 집중구조
  - 전사 분석업무를 별도의 분석전담조직에서 담당
  - 전략적 중요도에 따라 분석조직이 우선순위를 정해서 진행 가능
  - 현업 업무부서의 분석업무와 중복 및 이원화 가능성 높음
- 기능구조
  - 일반적인 분석 수행 구조로 별도 분석조직이 없고 해당 업무부서에서 분석 수행
  - 전사적 핵심분석이 어려우며 부서 현황 및 실적 통계 등 과거 실적에 국한된 분석
    수행 가능성이 높음
- 분산구조
  - 분석조직 인력들을 현업부서로 직접 배치하여 분석업무 수행
  - 전사차원의 우선순위를 선정해 수행하며 신속한 업무에 적합
  - 분석결과에 따른 신속한 피드백이 나오고 베스트 프랙티스 공유가 가능
  - 업무 과다와 이원화 가능성이 존재할 수 있기에 부서 분석업무와 역할 분담이 명확해야
    함

분석 과제 관리 프로세스 수립
- 분석 조직이 수행할 중요한 역할 중 하나가 분석 과제의 기획 및 운영이므로 이를 체계적
  으로 관리하기 위한 프로세스 수립 필요
- 분석 조직이 지속적이고 체계적인 분석관리 프로세스를 수행함으로써 조직 내 분석 문화
  내재화 및 경쟁력을 확보할 수 있음
- 분석 과제 관리 프로세스는 크게 과제 발굴과 과제수행 및 모니터링 단계로 나누어짐
- 과제 수행 단계에서는 팀 구성 후 분석 과제 실행, 분석 과제 진행 관리와 과제 결과를
  공유하고 개선하는 절차 수행
- 과제 발굴
  - 분석 idea 발굴 -> 분석 과제 후보 제안 -> 분석 과제 확정 -> 팀 구성
  - 개별 조직이나 개인이 도출한 아이디어를 발굴하고 이를 과제화하여 분석과제 풀로
    관리하면서 분석 프로젝트를 선정하는 작업 수행
  - 확정된 분석 과제를 풀로 관리하는 것이 아니라 분석 과제 중 발생한 시사점과 분석
    결과물을 풀로 관리
- 과제 수행 및 모니터링
  - 팀 구성-> 분석 과제 실행 -> 분석 과제 진행 관리 -> 결과 공유/개선
  - 분석을 수행할 팀을 구성하고 분석과제 실행 시 지속적인 모니터링과 과제 결과를 공유
    하고 개선하는 절차 수행

분석 교육 및 변화관리
- 모든 구성원이 데이터를 분석하고 업무에 즉시 활용할 수 있도록 조직 전반에 분석 문화를
  정착시키고 변화 되는 것이 중요
- 분석의 가치를 극대화하고 내재화하는 안정적인 추진기로 접어들기 위해서 분석에 관련된
  교육 및 마인드 육성을 위한 적극적인 변화 관리가 필요
- 분석 기획자, 분석 실무자, 업무 수행자 각각의 업무 역량을 강화하는 방향의 교육계획
  수립 필요
- 분석 기획자
  - 분석 큐레이션(목적에 따라 분류 및 기획) 교육
- 분석 실무자
  - 데이터 분석 기법 및 도구에 대한 교육
- 업무 수행자
  - 분석 기회 발굴, 구체화, 시나리오 작성법 교육
