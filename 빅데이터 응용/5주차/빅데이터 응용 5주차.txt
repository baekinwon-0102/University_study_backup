## 빅데이터 분석과 전략 인사이트

왜 싸이월드는 페이스북이 되지 못했나
- 데이터분석기반 경영문화의 부재에서 찾을 수 있음
- 비즈니스의 핵심 가치와 관련된 어떤 심도 있는 분석도 수행되지 않았음
- 소셜 네트워킹이 갖는 중요성에도 불구하고 회원들의 소셜 네트워킹 활동 특성과
  관련된 분석을 위한 프레임워크나 평가지표가 없었음

분석은 페이스북과 싸이월드의 운명을 가른 결정적 기준
- 페이스북은 서비스의 필요성을 파악하고 새로운 하나의 기능이 새로운
  수백만 페이지 뷰를 창조함
- 전략적 분석과 통찰력 창출에 눈을 돌려야 하는 이유가 이러한 이유 때문

빅데이터 분석, Big이 핵심이 아니다
- 많은 데이터가 많은 가치로 바로 연결 된다고 볼 수 없음
- 데이터의 양이 아니라 유형의 다양성과 관련있음
- 빅데이터가 가져다주는 기회는 크기보다 새롭고 다양한 정보 원천의 활용에 있음

전략적 통찰이 없는 분석의 함정
- 성과가 우수한 기업들도 가치 분석적 통찰력을 갖추었다고 대답한 비율이 낮음
- 자칫 일차적이고 부분적인 분석을 통해 손쉬운 해결책만 찾다가 전략적 통찰이 없는
  분석이 될 수 있음

각 산업에서 일반적으로 수행하는 일차적인 분석 활용 사례
- 금융 서비스: 신용점수 산정, 사기탐지, 클레임 분석, 고객 수익성 분석 등
- 소매업: 판촉, 수요예측, 매대관리 등
- 제조업: 공급사슬 최적화, 수요예측 등
- 운송업: 일정관리, 노선배정 등
- 헬스케어: 약품거래, 예비진단, 질병관리 등
- 병원: 가격책정, 고객 로열티 등
- 에너지: 에너지 트레이딩, 공급/수요예측 등
- 커뮤니케이션: 가격 계획 최적화, 고객 보유 등
- 서비스: 콜센터 직원관리, 서비스/수익 사슬 관리
- 정부: 사기 탐지, 범죄방지 등

## 전략 인사이트 도출을 위한 필요 역량

데이터 사이언스의 의미와 역할
- 데이터로부터 의미 있는 정보를 추출해내는 학문
- 통계학이 정형화된 실험데이터를 분석 대상으로 하는 것에 비해, 정형 또는 비정형을
  막론하고 다양한 유형의 데이터를 대상으로 함
- 분석뿐 아니라 이를 효과적으로 구현하고 전달하는 과정까지 포함한 개념
- 데이터 공학, 수학, 통계학 등 해당 분야의 전문 지식을 종합한 학문으로 정의

데이터 사이언스의 구성요소
- 데이터의 홍수 속에서 헤엄치고, 데이터 소스 찾고, 복잡한 대용량 데이터를 구조화하거나
  불완전한 데이터를 서로 연결해 결과 내놓음
- 패턴이나 통찰력있는 해결책을 찾아 결과 정보를 시각적으로 보여줌
- IT영역: 데이터 처리와 관련된 영역/ 시그널 프로세싱, 프로그래밍, 데이터 엔지니어링 등
- 분석영역: 수학, 머신러닝, 확률모델 분석학 등
- 비즈니스 컨설팅 영역: 커뮤니케이션, 프레젠테이션, 시각화, 스토리텔링 등

데이터 분야 직무
- 데이터 엔지니어
  - 비즈니스 이해하고 대량의 데이터 세트 가공
  - 제품을 최적화하기 위한 분석도구 개발
  - 하둡, 스파크 등을 이용해 댕요량 데이터 분산 처리 시스템 개발
- 데이터 분석가
  - 최적의 의사결정을 내리는데 도움을 주는 비즈니스 인사이트 제공
  - 데이터의 경향, 패턴, 이상값 등을 인식하기 위한 시각화 진행, 보고서 작성
  - 비즈니스 팀과 연계해 각 팀의 전략을 수립하거나 업무 효율화에 필요한 데이터 수집
    및 분석
- 데이터 사이언티스트
  - 머신러닝 모델을 사용해 정형, 비정형 데이터에서 인사이트 도출
  - 사내 데이터를 이용해 고객행동 패턴 모델링 진행, 패턴을 찾아내거나 이상치 탐지
  - 예측 모델링, 추천 시스템 등을 개발해 비즈니스 의사결정에 필요한 인사이트 제공
- 데이터 아키텍트
  - 데이터베이스를 쉽게 통합, 중앙 집중화 및 보호할 수 있도록 기업의 데이터 관리를 위한
   청사진을 만드는 업무
  - 효율성 및 보안을 고려한 기업의 데이터 아키텍처 계획 및 관리

데이터 사이언티스트의 요구 역량
- 빅데이터에 대한 이론적 지식: 관련 기법에 대한 이해와 방법론 습득 (하드스킬)
- 분석 기술에 대학 숙련: 최적의 분석 설계 및 노하우 축적 (하드스킬)
- 통찰력 있는 분석: 창의적 사고, 호기심, 논리적 비판 (소프트스킬)
- 설득력 있는 전달: 스토리텔링, 비주얼라이제이션 (소프트스킬)
- 다분야 간 협력: 커뮤니케이션
- 분석모델링, 데이터관리, 소프트스킬, 비즈니스분석
- 하드스킬: 해당 업무를 수행하는데 필요한 실질적 기술
- 소프트스킬: 모든 직무에서 사용할 수 있는 기술

데이터 사이언스: 과학과 인문의 교차로
- 통찰력 있는 분석은 직관과 전략, 경영 프레임워크, 경험이 혼합되어야 하는 개념
- 스토리텔링, 커뮤니케이션, 창의력, 열정, 직관력, 비판적 시각, 글쓰기 능력, 대화 능력
  등이 필요하다고 강조
- 예술과 과학을 혼합해 의사결정

인문학 열품의 외부 환경적 요소
- 세계화의 변화: 단순 세계화인 컨버전스 시기 -> 복잡한 세계화인 디버전스 시기로 변화
- 비즈니스의 중심 이동: 비즈니스의 중심이 제품생산에서 서비스로 이동
- 경제와 산업의 논리 변화: 경제와 산업의 논리가 생산에서 시장 창조로 변화

데이터 사이언티스트에 요구되는 인문학적 사고의 특성과 역할
- 어떻게 해서 인문학은 우리가 그 지식으로 세상을 이해한다고 믿는가를 따져 물음을
   던지는 것
- 통찰력 있는 분석과 그 맥을 같이할 수 있고 계속해서 질문들 찾고, 검증 가능한 가설
  세우고, 실험을 거듭하는 것 등이 통찰력 있는 분석의 요체

분석으로 다룰 수 있는 핵심 문제
- 최고의 데이터사이언티스트는 정량분석이라는 과학과 인문학적 통찰에 근거한 
   합리적 추론을 탁월하게 조합

데이터 분석 모델링에서 인문학적 통찰력의 적용 사례
- 타고난 성향의 관점: 유전적 요소처럼 원래의 성향이 존재하고 인간은 이것을 기초로 나뉨
- 행동적 관점: 한 사람의 행동을 지속적으로 관찰해 그 행동을 보고 그 사람 판단
- 상황적 관점: 단순하게 그 사람의 행동만 관찰하지 않고 상황과 맥락을 고려
- 인간에 대한 이해는 데이터 사이언티스트가 마케팅 모델 개발할 때 중요한 가이드가
  될 수 있음

## 빅데이터 그리고 데이터 사이언스의 미래

빅데이터의 회의론을 넘어 가치 패러다임의 변화
- 첫 단계의 가치 패러다임은 아날로그의 세상을 어떻게 효과적으로 디지털화하는가가
  이 시대의 가치를 창출해 내는 원천이었음
- 다음은 연결 패러다임인데 새로운 시대에서 디지털화된 정보와 대상들은 서로 
  연결되기 시작하며 연결을 얼마나 효과적이고 효율적으로 제공해 주느냐가 이 시대의
  성패 가름
- 앞으로는 연결이 더 증가하고 극도로 복잡해질것임
- 향후에는 복잡한 연결을 얼마나 효과적이고 믿을만하게 관리해주는가의 이슈인
  에이전시가 키워드로 등장할 확률 높음
- 에이전트 기능을 제대로 수행하기 위해 반드시 필요한 것이 데이터 사이언스와 관련된
  능력임

데이터 사이언스의 한계와 인문학
- 데이터 사이언스의 한계
  - 분석 과정에서 인간의 해석이 개입되면서 다른 해석과 결론 내릴 수 있음
  - 아무리 정량적 분석이라도 모든 분석은 가정에 근거
- 데이터 사이언스와 인문학
  - 가정들과 현실의 불일치에 대해 지속적으로 고찰하고 분석 모델이 예측할 수 없는 위험
    식별하기 위해 현실세계를 관찰해야 함
  - 분석 모델의 한계를 넘기 위해 경험과 세상에 대한 통찰려과 함께 분석을 활용해야 함

## 분석 기획 방향성 도출

분석 기획의 개념
- 실제 분석을 수행하기 전 분석을 수행할 과제의 정의 및 의도한 결과 도출을 위한
  적절한 관리방안을 사전에 계획하는 일련의 작업
- 분석 기획은 어떠한 목표 달성 위해 어떠한 데이터를 가지고 어떤 방식으로 수행할지에
  대한 일련의 계획을 수립하는 작업, 성공적인 분석 결과 도출 위해 중요한 사전 작업임
- 해당 문제에 대한 전문적 역량, 수학 및 통계적 지식을 활용한 분석 역량, 분석 도구인
  데이터 및 프로그래밍 기술 역량에 대한 균형 잡힌 시각으로 방향성 및 계획 수립해야 함

분석 기획의 특징
- 수학/통계학적 지식 및 IT 기술 뿐만 아니라 해당 비즈니스에 대한 이해와 전문성을
  포함해 3가지 영역에 대한 고른 역량과 시각 요구
  - 정보 기술, 수학/통계학, 분야의 전문 지식
- 분석 주제는 분석의 대상 및 분석의 방법에 따라 4가지로 나눔
  - 최적화, 통찰, 해결안, 발견

과제 중심적 접근 방식
- 당면한 과제를 빠르게 해결
- 1차 목표: 과제 단위로 당면한 분석 주제 해결 (Speed / Test)
- 과제 유형: Quick-win 방식의 데이터 분석 통해 과제 단위로 진행되는 프로젝트 문제에
  대한 명확한 해결
- 접근 방식: 문제해결을 위한 단기적 접근 방식

장기적 마스터플랜 방식
- 1차 목표: 마스터플랜 단위로 지속적 분석문화 내재화 (Accuracy/ Deploy)
- 과제 유형: Long Term View 방식으로 분석 과제를 도출하고 해당 과제를 수행함으로써 
  지속해서 데이터 분석 문화 내재화
- 접근 방식: 문제 정의를 위한 중장기적인 마스터플랜 접근방식

분석 기획 시 고려사항
- 분석의 기본이 되는 가용 가능한 데이터 확보 필요
  - 유형에 대한 분석이 선행적으로 이루어져야 함
  - 정형데이터, 비정형데이터, 반정형데이터의 존재 유무 및 유형을 파악하는 것 필요
- 분석을 통해 가치가 창출될 수 있는 적절한 활용 방안과 활용 가능한 유스케이스 탐색 
  필요
  - 기존에 잘 구현되어서 활용되고 있는 유사 분석 시나리오 및 솔루션 있다면 이를 최대한
    활용하는 것이 중요
- 분석 수행 중 발생하는 장애요소들에 대한 사전 계획 수립 필요
  - 일회성 분석으로 그치지 않고 조직의 역량으로 내재화하기 위해 충분하고 계속된
    교육 및 활용 방안 등의 변화 관리를 고려해야 함
  - 분석 수행 시 문제없이 실행되던 분석 결과가 실제환경에서 문제없이 발생할 수 있도록
    조치해야 함
  - 비용 대비 효과를 고려한 적정한 비용 산정해야 함

## 분석 방법론

기업의 합리적인 의사결정을 가로막는 장애요소
- 합리적 데이터 기반의 의사결정을 위해 장애 요소를 제거해야 하고 데이터 분석을
  활용해야 함
- 고정관념: 어떤 집단이나 사회적 구성원들의 전형적 특징에 관한 신념
- 프레이밍 효과: 동일한 사건이나 상황임에도 불구하고 문제의 표현 방식에 따라
  개인의 판단이나 선택이 달라질 수 있는 현상
- 편향된 생각: 공정하지 못하고 한쪽으로 치우친 사고나 견해

분석 방법론의 구성 요소
- 절차
  - 데이터 분석 프로젝트 수행시 이루어지는 각 작업 단계의 체계
  - 활동의 순서 명시, 단계별 활동, 활동별 세부작업
- 방법
  - 단계별 수행해야 하는 작업에 대한 구체적 설명
  - 언제, 어떻게 뭉서을 작업하는지 기술
- 도구와 기법
  - 단계별로 작업 수행 시 소요되는 기술 및 방법
  - 기법별 구체적인 사용 표준 및 방법
- 템플릿과 산출물
  - 단계별로 만들어야 하는 템플릿과 산출물의 목록 및 양식

분석 방법론 모델
- 폭포수 모델
  - 단계를 순차적으로 진행하는 방법
  - 이전 단계가 완료되어야 다음 단계로 진행할 수 있는 하향식 진행
  - 문제점이 발견되면 이전 단계로 돌아가는 피드백 과정 수행될 수 있음
- 나선형 모델
  - 반복을 통해 점중적으로 개발하는 방법으로 처음 시도하는 프로젝트에 적용 용이
  - 반복에 대한 관리 체계를 효과적으로 갖추지 못한 경우 복잡도가 상승해 프로젝트
    진행 어려움
  - 프로토타입을 지속적으로 발전시켜 최종 소프트웨어에 이르게 하는 모델
- 프로토타입 모델
  - 개발하려는 시스템의 주요 기능을 시스템 개발 초기에 실제 운영할 모형을 만들어 평가
    후 전체 시스템으로 확장하여 구현하는 모델
  - 가설을 생성하고 프로토타입을 보고 완성시킨 결과물을 통해 가설 확인 가능

KDD 분석 방법론
- 데이터베이스에서 의미 있는 지식을 탐색하는 데이터마이닝부터, 기계학습, 인공지능, 
  패턴인식, 데이터 시각화 등에서 응용될 수 있는 구조를 가지고 있음
- 데이터로부터 목표 데이터 구성, 패턴 파악 등을 통해 지식을 생성
- 데이터 세트의 선택에 앞서 분석 대상의 비즈니스 도메인에 대한 이해와 프로젝트의 목표
  를 정확하게 설정해야 함
- 분석 절차는 데이터 세트 선택, 데이터 전처리, 데이터 변환, 데이터 마이닝, 데이터 마이닝
  결과 평가의 5가지 단계로 이루어짐

KDD 분석 방법론 5가지 단계
- 데이터 세트 선택
  - 데이터베이스 또는 원시 데이터에서 필요 데이터 선택
  - 필요시 추가적으로 데이터 세트 생성
  - 데이터 마이닝에 필요한 목표 데이터 구성
- 데이터 전처리
  - 데이터 세트에 포함된 잡음, 이상값, 결측값 등을 식별하고 필요시 제거하거나 의미있는
    데이터로 처리하는 데이터 정제작업 시행
- 데이터 변환
  - 분석 목적에 맞는 변수를 선택, 데이터 차원 축소, 데이터 정규화 등을 수행하는 단계
  - 데이터 마이닝이 효율적으로 적용될수 있도록 데이터 세트를 변경
- 데이터 마이닝
  - 분석 목적에 맞는 데이터 마이닝 기법과 알고리즘 선택
  - 데이터의 패턴을 찾거나 데이터 분류 또는 예측작업 수행
  - 필요에 따라 데이터 전처리, 변환 프로세스와 병행 가능
- 데이터 마이닝 결과 평가
  - 분석 결과에 대한 해석과 평가, 분석 목적과의 일치성을 확인하는 단계
  - 발견된 지식을 활용하기 위한 방안 찾음
  - 필요시 데이터 세트 선택부터 데이터 마이닝까지 프로세스의 반복 수행

CRISP-DM 분석 방법론의 개요
- 비즈니스 이해를 바탕으로 데이터를 분석하기 위한 6단계로 진행되는 데이터마이닝
  방법론
- 계층적 프로세스 모델

CRISP-DM 분석 방법론의 구성
- 최상위 레벨은 여러 개의 단계로 구성되고 각 단계는 일반화 태스크 포함
- 두번째 레벨인 일반화 태스크는 데이터 마이닝의 단일 프로세스를 완전하게 수행하는
  단위
- 세번째 레벨은 세분화 태스크로 일반화 태스크를 구체적으로 수행하는 레벨
- 마지막 레벨인 프로세스 실행은 데이터 마이닝을 위한 구체적인 실행을 포함

CRISP-DM 분석 방법론의 분석 절차
- 1단계 (업무 이해)
  - 비즈니스 관점에서 프로젝트의 목적과 요구사항을 이해
  - 도메인 지식을 데이터 분석을 위한 문제 정의로 변경하고 프로젝트 계획 수립
  - 업무 목적 및 상황 파악, 데이터마이닝 목표 설정, 프로젝트 계획 수립
- 2단계 (데이터 이해)
  - 분석을 위한 데이터 수집, 데이터 속성을 이해하기 위한 과정
  - 데이터 품질의 문제점을 식별하며 숨겨져 있는 인사이트를 발견하는 단계
  - 초기 데이터 수집, 데이터 탐색, 데이터 품질 확인
- 3단계 (데이터 준비)
  - 수집된 데이터에서 분석기법에 적합한 데이터 세트를 편성하는 단계
  - 데이터 준비에 많은 시간 소요
  - 분석용 데이터 세트 선택 및 편성, 데이터 정제, 데이터통합, 데이터 포맷팅
- 4단계 (모델링)
  - 다양한 모델링 기법과 알고리즘을 선택하고 파라미터 최적화
  - 모델링 단계를 통해 찾아낸 모델은 테스트를 통해 과적합 등의 문제 발견하고 대응 방안
    마련
  - 모델링 기법 선택, 모델 테스트 계획 설계, 모델 선택 및 평가 수행
- 5단계 (평가)
  - 모델링 단계에서 구현한 모델이 프로젝트 목적에 부합하는지 평가
  - 데이터마이닝의 결과 수용 여부를 최종적으로 판단하는 과정
  - 분석결과평가, 모델링 과정평가, 모델 적용성 평가 수행
- 6단계 (전개)
  - 완성된 모델을 업무에 적용하기 위한 계획 수립
  - 전개 계획 수립, 모니터링과 유지보수 계획 수립, 프로젝트 종료보고서 작성, 프로젝트
    리뷰 작성 등
