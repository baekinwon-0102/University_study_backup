- 데이터의 이해 -  ★중간고사 기출 확률 매우 높음

데이터
- 데이터는 추론과 추정의 근거를 이루는 사실
- 데이터를 단순한 객체로서의 가치 뿐 아니라 다른 객체와 상호관계 속에서
  가치를 갖는 것
- 데이터는 객관적 사실이라는 특성을 갖는 동시에 추론, 예측 등을 위한 근거로 가능
- 정성적 데이터: 언어, 문자등으로 표시
- 정량적 데이터: 수치, 도형, 기호 등으로 표시
- 데이터는 암묵지와 형식지의 상호작용에 있어 중요한 역할 수행

암묵지
- 학습과 체험을 통해 개인에게 습득되어 있지만 겉으로 드러나지 않는 무형의 지식
   ex) 자전거 타기
- 개인의 암묵지를 언어나 기호 등의 형태로 표출화하고 이를 다른 개인이 본인의 지식에
   연결해 새로운 경험 부가하는 내면화 과정이 조직의 지식을 증가한다 봄
- 암묵지를 형식지로 변환 위한 데이터는 지식 형성의 중요한 기초 이룸

형식지
- 형상화된 지식 ex) 매뉴얼

암묵지와 형식지의 상호작용
- 1. 공통화 : 다른 사람과 대화 등 상호 작용을 통해 개인이 암묵지 습득
- 2. 표출화 : 개인에게 내재된 경험을 객관적 데이터인 문서나 매체로 저장, 가공, 분석
- 3. 연결화 : 형식지가 상호 결합하면서 새로운 형식지 창출
- 4. 내면화 : 행동과 실천 교육등을 통해 형식지가 개인의 암묵지로 체화

지식정보사회
- 데이터, 정보, 지식과 같은 용어를 흔하게 접함
- DIKW 피라미드는 데이터, 정보, 지식을 통해 최종적으로 지혜를 얻어내는 과정을
  계층적으로 설명

데이터 비식별화
- 특정 개인을 식별할 수 없도록 개인정보의 일부 또는 전부를 변환
- 데이터 안정하게 활용 위해 수집된 데이터의 개인정보 일부 또는 전부 삭제 또는
  다른 정보로 대체해 다른 정보와 결합해도 특정 개인 식별하기 어렵게 함

데이터 비식별화 처리기법
- 가명처리
  - 개인식별 가능한 데이터에 대해 식별할 수 없는 다른 값으로 대체
  - 자체로 완전 비식별화 가능, 데이터의 변형과 변질 수준 낮음
  - 일반화된 대체 값으로 가명처리해 성명을 기준으로 하는 분석에 한계가 존재
- 총계처리
  - 개인정보에 통계값을 적용해 특정개인 판단할 수 없도록 함
  - 집계 처리된 데이터로 정밀한 분석이 어렵고 집계한 데이터가 작은 경우
     결합 과정에서 개인정보 유출 또는 예측 가능
- 데이터값 삭제
  - 개인정보 식별 가능한 특정 데이터값 삭제
  - 분석의 다양성, 분석결과 유효성, 신뢰성을 저하시킬 수 있음
- 범주화
  - 단일식별 정보를 해당 그룹의 대표값으로 범주화해 고유정보 식별 금지
  - 범주나 범위는 통계형 데이터 형식으로 다양한 분석 가능
  - 데이터 범위의 구간이 좁혀지면 추적, 예측 가능
- 데이터 마스킹
  - 개인식별 정보에 대해 전체 또는 부분적으로 대체값(공백, *)으로 변환
  - 과도한 마스킹 적용시 필요한 정보로 활용하기 어렵고 마스킹 수준 낮으면
     특정 값 예측 가능

- 데이터베이스의 개요 -

데이터베이스
- 데이터베이스 용어가 처음 등장한 것은 1950년대 미국정부가 컴퓨터 기술로 구현한
  도서관을 설립하면서 비롯됨
- 이 때, 수집된 자료를 일컫는 데이터의 기지라는 뜻으로 데이터베이스 탄생
- 1963년 C. 바크만은 최초의 현대적 의미의 데이터베이스 관리시스템인
  IDS(integrated Data Store)를 개발해 저장 장치에 직접 접근 위한 데이터 모델 제안

데이터베이스 정의
- 체계적이거나 조직적으로 정리되고 전자식 또는 기타 수단으로 개별적으로 접근 가능한
  독립된 저작물, 데이터 또는 기타 소재의 수집물
- 동시에 복수의 적용 업무 지원할 수 있도록 복수 이용자의 요구에 대응해 데이터를
  저장, 공급 위해 일정한 구조에 따라 편성된 데이터 집합
- 체계적으로 정렬된 데이터 집합, 데이터량과 이용이 늘어나며 대용량 데이터를
  관리할 수 있는 컴퓨터 기반 데이터베이스로 진화

메타 데이터
- 데이터에 관한 구조화된 데이터로 다른 데이터를 설명해 주는 데이터

데이터베이스 특징
- 통합된 데이터: 동일한 내용의 데이터가 중복되어 있지 않음
- 저장된 데이터: 컴퓨터가 접근할 수 있는 저장 매체에 저장됨, 기본적으로 
  컴퓨터 기술 바탕임
- 공용 데이터: 여러 사용자가 서로 다른 목적으로 데이터베이스의 데이터를 공동으로 이용
- 변화되는 데이터: 새로운 데이터 삽입, 기존 데이터 삭제, 갱신으로 항상 변화하면서도
  현재 정확한 데이터 유지

데이터베이스의 특성
- 정보의 축적 및 전달
  - 대량의 정보를 일정한 형식에 따라 정보처리기기가 읽고 쓸 수 있게 하는
    기계가독성 있음
  - 다양한 방법으로 필요한 정보를 검색할 수 있는 검색가능성 있음
  - 정보통신망을 통해 원거리에서도 온라인으로 이용할 수 있는 원격조작성 있음
- 정보 이용
  - 이용자의 정보 요구에 따라 다양한 정보를 신속하게 휙득 가능
  - 원하는 정보를 정확하고 경제적으로 검색 가능
- 정보 관리
  - 정보를 일정한 질서와 구조에 따라 정리, 저장, 검색, 관리할 수 있게 함
  - 대량의 정보를 체계적으로 축적하고 새로운 내용을 추가, 갱신하기 용이
- 정보기술 발전
  - 정보처리, 검색/관리 소프트웨어 등 네트워크 기술 등의 발전 견인
- 경제 / 산업
  - 다양한 정보를 필요에 따라 신속하게 제공, 이용할 수 있음
  - 경제, 산업, 사회 활동의 효율성 높이고 국민의 편의를 증진하는 수단

데이터베이스 장단점
- 장점
  - 데이터의 중복 최소화
  - 데이터의 공유
  - 데이터의 일관성, 무결성, 보안성 유지
  - 최신의 데이터 유지
  - 데이터의 표준화 기능 제공
  - 데이터의 논리적, 물리적 독립성
  - 쉬운 데이터 접근
  - 데이터 저장 공간 절약
- 단점
  - 데이터베이스 전문가 필요
  - 관리 비용 부담
  - 데이터 백업과 복구 어려움
  - 시스템 복잡함
  - 대용량 디스크로 액세스 집중되면 과부하 발생
  - 통합된 시스템으로 일부에서 장애 발생 시 전체 시스템 중단되는 장애 발생

데이터베이스 관리 시스템(DBMS)
- 데이터베이스 관리시스템은 다수 사용자가 데이터베이스에 접근해 데이터베이스의 
  정의, 조작, 제어 등 데이터베이스 관리를 지원하는 소프트웨어
- 관계형 DBMS
  - 데이터 모델이 간단해 이해 쉽고 데이터 저장 위치와 접근방법은 DBMS에서 결정
  - 행과 열로 구성된 2차원 테이블에 데이터 저장
- 계층형 DBMS
  - 데이터를 레코드와 링크로 구성된 트리 형태로 표현
  - 부모와 자식 노드는 일 대 다 관계(1:N)로 구성
- 네트워크형 DBMS
  - 그래프 구조에 기반을 두며 레코드를 노드로, 레코드와 레코드 간 관계는 간선으로 표현
  - 레코드와 레코드 간 관계는 다 대 다 관계로 표현
- 객체지향형 DBMS
  - 객체지향 프로그래밍에 사용하는 것으로 정보를 객체의 형태로 표현하며
    일부 전문 분야에 사용
  - 복잡한 데이터 구조의 정보 모델링 가능하고 사용자 정의 타입, 비정형 정보 타입 지원

SQL
- 데이터베이스 관리시스템에서 자료의 검색과 관리를 위해 SQL 언어 사용
- 자료의 검색과 관리, 스키마 생성 및 수정, 객체 접근 조정 관리를 위한 프로그래밍 언어
- 스키마 구조 제어 및 데이터베이스 테이블 데이터 조회를 위해
  데이터 정의어, 조작어, 제어어 사용
- 스키마: 데이터베이스의 구조와 제약 조건에 관한 전반적 명세를 기술한 데이터 집합

데이터 정의어
- 테이블과 같은 데이터 구조를 정의하는데 사용하는 명령어
- CREATE: 테이블 생성
- ALTER: 테이블 수정
- DROP: 테이블 삭제
-RENAME: 테이블 이름 변경

데이터 조작어
- 데이터베이스에 저장된 자료 입력, 수정, 삭제, 조회하는 명령어
- SELECT: 테이블의 데이터 조회
- INSERT: 테이블에 데이터 삽입
- UPDATE: 테이블의 데이터 변경
- DELETE: 테이블의 데이터 삭제

데이터 제어어
- 데이터베이스 관리자가 데이터 보안, 무결성 유지, 병행 제어, 회복을 위해 관리자가
  사용하는 제어용 언어
- GRANT: 관리자가 사용자에게 데이터베이스 권한 부여
- REVOKE: 관리자가 사용자에게 부여했던 권한 회수

SELECT문
- SELECT 절
  - 검색하고자 하는 속성, 계산식 기술
  - 2개 이상 테이블을 대상으로 검색할 때 '테이블명.속성명'으로 표현
  - SELECT 절 뒤에 명시하지 않을 경우 ALL로 인식
  - 중복된 속성이 조회될 경우 그 중 한 개만 검색
- FROM 절
  - 질의에 의해 검색될 데이터들을 포함하는 테이블명 기술
- WHERE 절
  - 검색할 조건 기술
- GROUP BY 절
  - 속성값을 그룹으로 분류하고자 할 때 사용
- HAVING 절
  - GROUP BY에 의해 분류한 후 그룹에 대한 조건 지정
- ORDER BY 절
  - 속성값 정렬하고자 할 때 사용

-데이터베이스 활용-

기업내부 데이터베이스
- 정보통신망 구축이 가속화되면서 1990년부터 기업내부 데이터베이스는 기업 경영 전반에
  관한 인사, 조직, 생산, 영업활동을 포함한 모든 자료를 연계해 일관된 체계로 구축, 운영
  하는 경영활동의 기반이 되는 전사 시스템으로 확대됨
- 이를 OLTP(Online Transaction Processing) 시스템으로 볼 수 있음
- 1990년대 중반 기점으로 데이터 마이닝 등의 기술이 등장하면서 단순한 정보 수집에서
  탈피해 분석이 중심이 된느 시스템 구축으로 변화
- 이를 OLATP(Online Analytical Processing) 시스템으로 볼 수 있음
- 2000년대 들어 기업 데이터베이스 구축의 화두는 CRM(고객관계관리), SCM(공급망관리)
  로 바뀜
- CRM은 고객별 구매이력 데이터베이스를 분석해 각종 마케팅 전략을 펼치는 것을 말함
- SCM은 외부 공급업체 또는 협력업체와 통합된 정보시스템을 이용해 시간과
  비용을 최적화하기 위한 관리 방법

제조부분
- 2000년 이전 부품이나 재고관리 등의 영역에서 데이터베이스 활요이 중점 이룸
  2000년 이후 부품의 설계, 제조, 유통의 전 공정을 포함하는 범위로 확대
- 2000년 중반 이후에는 중소기업에 대한 인하우스 DB 구축 투자의 증가가 있었음
  실시간 기업(RTE)의 개념이 화두가 됨
- RTE는 기업의 비즈니스 프로세스를 투명하고 민첩하게 유지해 환경 변화에 따른
  적응 속도를 최대하하여 지연시간을 없애는 정보화 전략

제조부분 기업 내부 데이터베이스 솔루션
- ERP(전사적 자원관리)
  - 회사의 모든 정보 뿐 아니라 공급망 관리, 고객의 주문정보까지 포함해 통합적으로 관리
- 데이터 마트
  - 전사적으로 구축된 데이터 속 특정 주제, 부서 중심으로 구축된 작은 규모의 
    데이터 웨어하우스
  - 데이터 마트는 데이터 웨어하우스의 일부로 대체로 특정 조직 및 팀 등 제한된 사용자
    그룹에 서비스 제공
- BI(Business Intelligence)
  - 비즈니스를 운영할 때 나오는 데이터를 수집, 저장, 통합 및 분석해 기업 활동과
    연관된 의사결정 돕는 프로세스
- DW(데이터 웨어하우스)
  - 사용자의 의사결정에 도움을 주기 위해 데이터베이스 축적된 데이터를 공통 형식으로
    변환하여 관리하는 읽기 전용 데이터베이스
  - 주체지향적: 기능이나 업무가 아닌 주제중심적으로 구성
  - 통합적: 데이터의 일관성 유지하면서 전사적 관점에서 하나로 통합
  - 시계열적: 시간에 따른 변경을 항상 반영
  - 비휘발적: 적재가 완료되면 읽기 전용 형태의 스냅샷 형태로 존재

금융부분
- 2000년 초반에 EAI(Enterprise Application Integration), ERP, e-CRM등과 같이
  데이터베이스간 정보 공유 및 통합이나 고객정보의 전략적 활용이 주된 이슈
- 2000년 중반에는 데이터웨어하우스를 적극 도입해 관련 마케팅을 증대시키기 위한
  노력이 가시화 됨
- 데이터웨어하우스는 데이터베이스에서 의사결정에 필요한 데이터를 미리 추출해 이를
  원하는 형태로 변환하고 통합한 읽기 전용의 데이터 저장소

금융부분 기업 내부 데이터베이스 솔루션
- EAI
  - 기업에서 운영하는 서로 다른 기종의 애플리케이션 및 시스템을 통합하는 솔루션
- ERP
  - 회사의 모든 정보뿐만 아니라 공급망 관리, 고객의 주문 정보까지 포함해 통합적으로
     관리하는 시스템
- e-CRM
  - 온라인상에서 고객의 행동과 성향을 분석해 고객 만족을 극대화하고 실시간 원 투 원
    마케팅을 실현해 주는 것으로 인터넷을 이용한 통합 마케팅 방법
- EDW
  - 여러 애플리케이션의 비즈니스 정보를 중앙 집중화해 조직 전체해서 분석 및 사용할 수
    있도록 함
- 블록 체인
  - 관리 대상이 된느 데이터를 P2P 방식 기반으로 생성된 연결 형태의 연결고리 기반의
    분산 데이터 저장환경에 저장해 누구도 임의로 수정할 수 없고 누구나 변경된 결과
    열람할 수 있는 분산 컴퓨팅 기술 기반 관리 기술

유통 부분
- 2000년 이후 전반적인 IT 변화 환경에 맞물려 CRM과 SCM 구축 이뤄짐
- CRM: 고객별 구매이력 데이터베이스 분석하고 고객에 대한 이해를 도우며 이를 바탕으로
 각종 마케팅 전략 펼침
- SCM: 기업이 외부 공급어체 또는 제휴업체와 통합된 정보시스템으로 연계해 시간과 비용
  최적화
- KMS: 조직 내의 인적자원이 가지고 있는 개별적 지식을 체계화해 공유함으로써 기업의
  경쟁력을 강화하기 위한 기업정보 시스템
- 2000년대 중반 이후 체계적인 고객정보의 수집, 분석과 상권분석 등으로 심화되었고
  균형성과 관리, 핵심성과지표 등 다양한 고객 분석 도구를 통해 기존 데이터베이스와 연계

사회 기반구조로서의 데이터베이스
- 1990년 이후 정부부처 중심으로 무역, 통관, 조세, 국세, 조달 등 전자문서교환 활용이
  본격화되면서 부가가치통신망(VAN: Value Added Network)을 통한 정보망 구축
- 정보망을 통해 유통되거나 이용되는 정보가 데이터베이스로 구축되었고 이는 공공
   DB로서 활용되기도 하고 관련 산업에서 유용하게 활용
- 1990년대 후반에는 데이터베이스 활용이 문서교환 등에 국한되지 않고 국가적으로
  필요한 기반시설인 지리, 교통부문의 데이터베이스 구축 시작
- 2000년대 들어서 의료, 교육, 행정 등 사회 각 부문으로 공공 DB 구축 확대

물류 부분
- 1995년 국가기간전산망사업으로 확정되어 출범한 종합물류정보망은 1998년에 이르러
  서비스 개발을 완료해 사용서비스로 제공되어 전자지도상 운행중인 차량의 위치 및 
   상태를 실시간으로 파악해 관제함으로써 운송회사 및 화주 등 서비스 기업자의 합리적
   의사결정 지원
- 2003년 이후 종합물류정보망의 이용 확대와 활성화를 위한 내륙화물기지와 같은 주요
  물류거점 시설의 지속적인 정보화 추진해 물류정보 공동활용 체계 확대

지리부분
- NGIS: 전 국토의 지상과 지하에 있는 시설물과 자원 같은 현황 정보를 수치 지도의 형태로
  제작하여 데이터베이스로 구축
- GPS: 위성에서 나오는 위치 분석해 사용자의 현재 위치를 계산하는 위성 항법 시스템

교통부분
- 2001년에 5개 광역시별 통행량, 대중교통 이용실태, 교통유발원 단위 및 주요구간 교통량
  조사로 질 높은 교통 데이터베이스 서비스가 이뤄짐
- 2002년부터 전국 교통 데이터베이스 구축 사업이 국가 교통 데이터베이스 구축 사업으로
  변경되어 활용 단계 돌입
- 2003~2007년에는 2단계 국가교통 데이터베이스 구축 사업을 통해 각종 예산절감과 
  사업기간 단축, 신뢰성 제고의 효과 거둠
- ITS: 자동차와 같은 교통수단, 도로와 같은 교통시설에 첨단 기술 적용해 차량 흐름을
  개선하고 차량 안전을 증진하기 위한 정보통신의 기술 조합

의료 부분
- 1990년대부터 의료정보망이 구축되면서 의료전자문서 교환 상용서비스가 제공되기 시작
- 2002년 의료법 개정을 통해 전자의무기록이 법적 효력을 갖게 되고 원격 의료와
  전자처방전이 허용되면서 의료정보시스템이 고도화
- 2005년 이후 유비쿼터스(언제 어디서나 접속 가능한 정보통신 환경)컴퓨팅 기술의 발달로
  의료정보 분야에서도 유비쿼터스 헬스 시장이 본격적으로 등장

교육 부분
- 대학정보화 관련해 대학도서관 소장자료의 데이터베이스 구축이 활발히 추진되어 서지 및
  기사색인을 비롯해 2003년부터 원문까지 데이터베이스로 구축
- 2000년 중반에 접어들어 모바일 환경이 확산되면서 모바일 캠퍼스 구축 등에 대한 투자
  확대
- 2003년부터 적용되기 시작한 교육행정정보시스템(NEIS)은 과거 학교별로 데이터베이스
  구축해 운영하던 학생, 교원 관렬 교육정보를 각 시, 도 교육청 데이터베이스에 담아놓고
  교무, 학사뿐 아니라 인사,물품,회계 등 기타 교육행정 전 업무를 처리하는 시스템임
