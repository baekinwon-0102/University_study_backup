Hadoop

- 데이터 분산 저장/ 분산 처리 플랫폼
- 상용화된 일반적인 컴퓨터 클러스터( 일반적인 컴퓨터의 집합) 에서 매우 큰 용량의 데이터를 분산 처리, 저장하기 위해 개발
- 분산된 데이터는 각 컴퓨터에서 처리됨
//Hadoop 장점
- 저렴한 비용 -> 상용화된 컴퓨터로 가능
- 안전함 -> 분산저장
- 빠름  -> 분산처리
- 확장성 -> 상용화된 컴퓨터
- 다양한 데이터 유형 지원
- 다양한 도구 및 프레임워크 지원

//Hadoop 단점
- 작은 크기 파일은 비효율적 ( 네트워크로 통신해야 하기 때문에 작은 파일은 로컬로 하는게 나음 )
- 높은 지연시간 : 실시간 응답에 약함 -> 이것도 네트워크 통신 때문에 로컬로 하는게 나음
  => 요걸 해결한게 Spark 
- 복잡한 설치 및 운영
- concept: wirte once, read many -> 수정/삭제가 까다롭고 성능이 낮음
- 보안 설정이 까다로움 -> 하드웨어 공유로 성능을 높였기 때문에 데이터 접근이 쉬움

Software platform

- 컴퓨터 클러스터에 의해 작동하는 소프트웨어 묶음

Distributed storage

- 하나의 저장소에 국한되지 않고 클러스터에 컴퓨터를 추가하면 해당 컴퓨터의
  저장장치가 데이터 저장소 일부가 됨
  ★ 클러스터의 모든 저장소에 분산된 데이터를 단일 파일 시스템으로 보여줌
  ★ 백업 복사본을 클러스터의 다른 컴퓨터에도 보관

Distributed processing

- 클러스터에 속한 컴퓨터들이 함께 데이터 처리 ( 병렬 처리 )
- Haddop은 클러스터 내 모든 컴퓨터 CPU에게 작업 분배
